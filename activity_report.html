Warehouse Pro: Sales & Inventory Lifecycle

sequenceDiagram participant User as Sales Executive participant Sys as Warehouse System participant DB as Database participant Email as SMTP Server participant Audit as Audit Logger User->>Sys: initiate Sale (Select Product) Sys->>DB: query Stock Level & HSN DB-->>Sys: Return SKU Status alt Out of Stock Sys-->>User: Show Stock Deficiency Error else In Stock Sys->>Sys: Calculate GST (CGST/SGST/IGST) Sys->>User: Present Order Summary User->>Sys: Confirm Payment (Cash/UPI/Card) Sys->>DB: Process Transaction DB->>DB: Update Stock: Stock = Stock - Qty DB->>DB: Record Sale & Invoice Details Sys->>Audit: Log "New Sale Finalized" (id, amount) Sys->>DB: Update User XP (+50 XP) par Async Processes Sys->>Email: SMTP Send: PDF Invoice to Customer Sys->>Sys: Trigger "Low Stock" Alert if reorder < level end Sys-->>User: Sale Success + Redirect to Invoice end

Technical Process Detail

  • Concurrency: Stock deduction and sale recording are handled in a single SQL transaction to maintain data integrity.
  • Auditability: Every sale triggers a background event that logs the user ID, timestamp, and transaction ID for compliance.
  • Asynchronous Tasks: Invoice delivery via SMTP and reorder predictions are decoupled from the main UI thread to ensure high performance at checkout.
